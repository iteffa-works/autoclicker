## Что должно уметь приложение

### 1. Автокликер
Реализовать:
- ЛКМ / ПКМ / СКМ
- одинарный клик
- двойной клик
- удержание кнопки мыши
- настраиваемый CPS / интервал в миллисекундах
- случайный разброс задержки (humanized jitter) в разумных пределах
- режим клика:
  - по текущей позиции курсора
  - по сохраненной позиции X/Y
- отображение текущих координат курсора
- кнопки:
  - Старт
  - Пауза
  - Стоп
  - Сброс

### 2. Бинды клавиш
Сделать систему биндов:
- возможность назначать горячие клавиши на:
  - старт кликера
  - стоп кликера
  - паузу
  - запись макроса
  - запуск макроса
  - экстренную остановку
- поддержка одиночных клавиш и комбинаций:
  - F1–F12
  - Ctrl, Alt, Shift
  - буквенные клавиши
  - цифровые клавиши
  - numpad
  - стрелки
  - Esc, Tab, Enter, Space, Backspace
  - Insert, Delete, Home, End, PgUp, PgDn
- конфликтующие бинды должны валидироваться
- должна быть кнопка “Слушать клавишу” для удобной привязки
- сохранить бинды между запусками приложения

### 3. Макросы
Нужен полноценный модуль макросов:
- запись последовательности действий пользователя:
  - нажатия клавиш
  - отпускания клавиш
  - клики мыши
  - движение мыши (по желанию, отдельной опцией)
  - прокрутка колесика
- возможность сохранить макрос
- возможность загрузить макрос
- список макросов в интерфейсе
- редактирование названия макроса
- удаление макроса
- повтор воспроизведения:
  - 1 раз
  - N раз
  - бесконечный цикл до остановки
- настройка задержек:
  - исходные
  - ускоренные
  - замедленные
  - пользовательский множитель скорости
- возможность редактирования шагов макроса в таблице:
  - тип события
  - клавиша/кнопка
  - задержка
  - координаты если мышь
- экспорт/импорт макросов в JSON

### 4. Поддержка нескольких клавиш и комбинаций
Нужно поддерживать:
- нажатие нескольких клавиш подряд
- комбинации типа Ctrl+C, Ctrl+Shift+S, Alt+Tab
- последовательности из нескольких шагов
- макросы смешанного типа: клавиатура + мышь
- корректную обработку модификаторов

### 5. Универсальные горячие клавиши
Реализовать глобальные горячие клавиши:
- должны работать даже если окно приложения не в фокусе
- обязательно сделать:
  - Start hotkey
  - Stop hotkey
  - Pause hotkey
  - Emergency stop hotkey
- если библиотека ограничена, предусмотреть fallback
- все горячие клавиши должны безопасно отключаться при выходе из приложения

### 6. Визуальная клавиатура для тестирования
Сделать отдельную вкладку “Тест клавиатуры”:
- визуальная клавиатура в интерфейсе
- подсветка нажатых клавиш в реальном времени
- отображение:
  - обычных клавиш
  - функциональных клавиш
  - стрелок
  - numpad
  - модификаторов
- отдельный блок “Тест мыши”:
  - индикация ЛКМ / ПКМ / СКМ
  - текущие координаты мыши
  - индикация прокрутки
- режим тестирования должен помогать проверять бинды и макросы

### 7. Настройки
Добавить полноценные настройки:
- тема интерфейса: светлая / темная
- язык интерфейса: задел под локализацию
- запуск поверх всех окон (опция)
- звук при старте/стопе
- минимизация в трей
- сохранение последней конфигурации
- путь папки для макросов
- настройка логирования
- настройка jitter/random delay
- настройка точности таймингов

### 8. Производительность и тайминги
Сделай так, чтобы приложение:
- работало плавно
- не блокировало GUI
- использовало отдельные worker threads / QThread / thread-safe подход
- имело максимально стабильные тайминги в рамках возможностей Windows user-space
- использовало высокоточный таймер там, где это возможно
- не зависало при запуске/остановке макросов
- имело безопасную остановку циклов

Важно:
- не использовать скрытые драйверы
- не использовать kernel-mode
- не делать stealth-функции
- не делать обход защит
- приложение должно работать как обычный пользовательский automation tool через стандартные публичные API

### 9. Безопасность и UX
Обязательно:
- явный видимый интерфейс
- явный индикатор “Активен / Пауза / Остановлен”
- emergency stop по умолчанию, например Esc или F12
- защита от случайного бесконечного запуска
- подтверждение при первом включении бесконечного цикла
- лог событий в нижней панели
- обработка исключений без падения приложения

### 10. Сохранение данных
Нужно сохранять:
- настройки
- бинды
- макросы
- последние координаты
- профили пользователя

Используй:
- JSON или SQLite
- желательно отдельный `config/` и `data/`
